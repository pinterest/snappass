Why do I have to go through this intermediary page instead of going directly to the password? Chat services like Facebook and Slack will issue a GET request the moment you paste a URL into their text box. With the original way snappass was designed, this renders the URL inaccessible and the end user would never see the password. This intermediary page solves that issue.
